# CognitiveDemo
这是一个问卷类型的项目。

frontend
前端需要时react 设计，可以自适应不同大小的屏幕（包括手机页面，mac电脑页面，显示器页面），每一个页面是一个选择题，单选。
等到最后一个选择题的时候，给后端发送一个请求，拿到结果。同时后端会带来一个token存在浏览器中，是cookie。
然后如果再次来到这个页面，如果有cookie的话，可以通过cookie发送查询请求，拿到结果。
我希望选项的界面有很漂亮的结果。
选项可以从一个json文件中加载

结果页面需要有准确与否以及赞赏的选项

可以设计成 两个连续的小动作：
第一步（极轻）——共鸣确认
比如一个按钮或选择：
「这份分析是否在整体上贴近你？」
很贴近
有部分贴近
不太贴近
这个是低心理负担的。
第二步（可选）——触点标记
只在用户愿意时出现：
「有没有哪一段特别击中了你？」
（可多选）
对我当前困境的描述
对我行为模式的解释
对我内在冲突的命名
对未来方向的提示
这里你收集到的是 “解释力来源”，非常宝贵。
还有赞赏

任何一个选项都会调用后端存入数据库。

backend
后端需要采用fastapi的设计。
有一个analyze的接口，用来拿到一系列选项后，去后端，先用prompt+选项调用大模型，然后从大模型获取结果。
不确定是用langchain还是langgraph 比较好
